# CSI PowerPoint Skill

## Purpose
Create, review, and revise CSI PowerPoint presentations so they are brand-compliant, readable, concise, and executive-ready.

## Source Assets
Use the latest approved files from `M:\ARS\02_Presentations\Brand Guidelines`.
Required files: `2025-CSI-Brand-Guidelines.pdf` and `2025-CSI-PPT-Template.pptx`.
When a current deck is provided, preserve its intent unless a rule violation requires changes.

## Modes
Use the mode requested by the user. If no mode is specified, default to `Audit + Fix Plan`.
`Create`: build a new deck from the CSI template.
`Audit`: review only; do not edit the deck.
`Fix`: revise the deck and return an edited version.
`Final QA`: run delivery-readiness checks; avoid creative changes unless violations remain.

## Priority Order
When rules conflict, follow this order:
1. Legal, compliance, confidentiality, and source requirements
2. CSI logo integrity and brand rules
3. Customer, partner, and co-branding permissions
4. CSI PowerPoint template structure
5. Slide clarity and readability
6. Copy polish and visual refinement

## Core Slide Rules
Apply these rules to all standard content slides:
1. Use the approved CSI PowerPoint template.
2. Use Montserrat for PowerPoint presentations.
3. Communicate one main idea per slide.
4. Use a one-line slide title.
5. Use 44 pt Montserrat ExtraBold for standard content-slide titles.
6. If a title wraps, rewrite it shorter. Do not shrink the title font.
7. Use no more than two primary font sizes per slide.
8. Use no more than three major points per slide.
9. Avoid dense paragraphs.
10. Move supporting detail into speaker notes.
11. Split crowded slides into multiple slides.
12. Preserve whitespace.
13. Keep alignment, spacing, and hierarchy clean.
14. Do not allow overlapping or off-slide objects.
15. Do not use unreadably small text.
16. Do not add decoration that does not clarify the message.

## Major Point Definition
A major point is any primary idea shown as a bullet, numbered item, icon callout, stat card, table row group, standalone text block, or labeled process step.
Standard content slides may contain no more than three major points.
If a slide has four or more major points, simplify, split, or move detail to speaker notes.

## Font Rules
Use Montserrat for PowerPoint slides.
Use the active CSI template body style for the selected layout.
Do not introduce extra primary font sizes.
Do not overuse bold, italic, underline, all caps, or color emphasis.
Use font weight and size to create clear hierarchy.

## Two-Font-Size Rule
The two-font-size rule applies to primary content only.
Footer text, page numbers, copyright text, confidentiality labels, source notes, legal disclaimers, chart labels, table source notes, and trademark symbols do not count as primary font sizes.

## Brand Color Rules
Use CSI Navy `#00274c` as the primary anchor color.
Use CSI Gold `#f8971d`, CSI Orange `#f15d22`, and CSI Red `#eb2a2e` as accents.
Use white `#ffffff` or light gray `#f8f8f8` for clean backgrounds when appropriate.
Use accent colors strategically and in moderation.
Do not combine large areas of multiple brand colors on the same slide.
Use tints and shades only when they support contrast and clarity.

## Logo Rules
Use the correct CSI logo version for the slide background.
Use the white-lettering logo on dark backgrounds.
Use the CSI Blue-lettering logo on light backgrounds.
Use the one-color logo when contrast or reproduction requires it.
Maintain clear space around the logo equal to the height of the “I” in the CSI wordmark.
Keep the logo legible and proportional.
Never stretch, squeeze, crop, rotate, skew, recolor, outline, or add effects to the logo.
Never separate the CSI symbol from the wordmark.
Never place the logo on busy, distracting, bright, or low-contrast backgrounds.
Logo-rule exceptions require written approval from CSI Marketing.

## Approved Layouts
Use approved CSI template layouts before creating custom layouts.
Approved slide types: title slide, presentation summary, section divider, photo with bullets, two-photo slide, color bar content slide, gray break slide, stats slide, icon list, map slide, additional resources, and questions/discussion.
Create a custom layout only when no approved layout supports the content.

## Content Rules
Write slide copy as short phrases, not paragraphs.
Use speaker notes for explanation, nuance, and backup detail.
Prefer visuals, diagrams, icons, charts, or impactful phrases over dense text.
Do not read the audience the slide.
Do not invent data, claims, dates, statistics, customer names, or product facts.
Flag unsupported claims for review.

## Voice Rules
Write in CSI’s brand voice: authentic, conversational, helpful, and confident.
Be clear, concise, engaging, appropriate, respectful, useful, and direct.
Use active voice.
Use “you” and “your” when appropriate.
Focus on customer outcomes.
Avoid jargon, vague claims, generic phrases, passive voice, overselling, bragging, and feature dumping.
Use strong verbs such as “helps” or “empowers” instead of weak phrasing such as “can help” when accurate.

## Copy Style Rules
Use AP style where applicable.
Use one space after periods.
Use trademark symbols on first mention of registered or trademarked CSI product names.
Use sentence case for clear CTAs.
Use specific CTAs instead of generic “Learn More” when possible.
Use CSI-preferred spellings: cybersecurity, cybercriminal, cyberattack, fintech, website, webpage, email, watch list, data center, eSignature, blockchain, cryptocurrency.
Use `real-time` as an adjective and `real time` as a noun.
Use `on-demand` as an adjective and `on demand` as a noun.

## Icon Rules
Use approved icons from the CSI PowerPoint template.
Use icons only when they clarify the point.
Keep icon style, weight, scale, and color treatment consistent.
Use CSI-approved colors only.
Do not mix unrelated icon styles.
Do not use icons as decoration without meaning.

## Photography Rules
Use photos with depth, perspective, crisp focus, high contrast, and modern energy.
Prefer original CSI product or brand photography when available.
Use photos with interesting angles or modern elements.
Ensure image and video usage rights are cleared.
Do not use low-quality, generic, distorted, or off-brand imagery.

## Chart and Data Rules
Each chart or stat slide must communicate one takeaway.
Use simple charts with clear labels.
Avoid 3D charts, unnecessary legends, and chart junk.
Use CSI colors with restraint.
Include source and date for external data.
Move complex tables to the appendix when they cannot be read comfortably.
Do not change numbers or statistics unless the user provides corrected data.

## Accessibility Rules
Ensure strong text/background contrast.
Do not rely on color alone to communicate meaning.
Use readable font sizes.
Add alt text for meaningful visuals when accessibility is required.
Avoid crowded layouts that reduce comprehension.

## Exception Policy
Apply all core rules to standard content slides.
Exceptions are allowed only when they improve clarity, preserve brand integrity, or satisfy legal, source, partner, customer, or template requirements.

### No-Exception Rules
Do not alter CSI logo color, proportions, spacing, orientation, or component order.
Do not ignore logo clear space.
Do not use the wrong logo version for the background.
Do not use non-CSI colors as dominant brand colors.
Do not use unapproved co-branding.
Do not change legal language, disclaimers, customer quotes, or third-party logos without explicit approval.

### Controlled Exceptions
These slide types may bend title, font-size, or point-count rules when needed: cover slides, section dividers, Q&A or closing slides, agenda slides, appendix/reference slides, legal/disclosure slides, source-heavy slides, chart/table/map/data slides, product screenshot or UI walkthrough slides, and customer/partner-provided slides.
Agenda slides may contain more than three agenda items.
Appendix/reference slides may contain more than three items if clearly marked.
Legal/disclaimer slides may contain required text.
Resource slides may contain grouped links, icons, or contact details.
Chart/table slides may contain supporting labels, but still need one main takeaway.

### Customer or Partner Slides
Convert customer or partner slides into CSI style when possible.
If third-party content cannot be modified, place it in an appendix or clearly marked provided-content section.
Do not alter third-party logos.
Do not imply CSI endorsement beyond the approved relationship.
Use CSI co-branding rules when both brands have ownership.

## Exception Documentation
When an exception is used, add this speaker-note entry:
`Exception used: [type]`
`Reason: [why needed]`
`Action taken: [what changed]`
`Review needed: Yes/No`

Flag for human review when an exception affects CSI logo usage, co-branding, customer or partner branding, legal language, non-approved colors, external statistics, or image/video usage rights.

## Audit Report Format
When auditing a deck, return:
- Executive summary
- Slide-by-slide findings
- Pass, warning, or fail status
- Severity: critical, major, or minor
- Recommended fix
- Exception used or needed
- Human-review items

## Fix Report Format
When revising a deck, return:
- Edited deck file
- Summary of major changes
- Slides split or consolidated
- Titles rewritten
- Content moved to speaker notes
- Exceptions documented
- Human-review items

## Final QA Checklist
Before final delivery, check every slide for:
- Approved CSI template use
- One clear message
- One-line title on standard slides
- 44 pt standard title treatment
- No more than two primary font sizes
- No more than three major points
- Correct CSI colors
- Correct logo version and clear space
- Clean alignment and whitespace
- No dense paragraphs
- No overlaps or off-slide objects
- Readable contrast
- Consistent icon and image style
- Source/date for external data
- Speaker notes for excess detail
- CSI voice and copy style
- Required exceptions documented
- Human-review items flagged

## Delivery Behavior
Never claim a deck is fully approved by CSI Marketing, Legal, Compliance, or a customer.
Say the deck is “ready for review” unless the user confirms approval.
When uncertain, preserve the original content and flag the issue instead of guessing.
