# How to Add a New Section

## Quick Start (Copy-Paste Template)

Create a new file in `02_Powerpoint/sections/`. Copy this template:

```python
"""<Section Name> -- <One-Line Question This Section Answers>

Mirrors: 01_Analysis/00-Scripts/analytics/<folder>/
Slide IDs: <list the IDs this section owns>
"""

from __future__ import annotations

from ._base import LAYOUT_CUSTOM, SectionSpec

# Prefixes: how slide IDs are matched to this section.
# The assembler looks at the part before the first "-" or "." in the slide ID.
# Example: slide "A8.3" has prefix "a8", slide "DCTR-5" has prefix "dctr".
_PREFIXES = ["a8", "rege"]

# Layout map: slide_id -> (layout_index, slide_type)
# See LAYOUT CHEAT SHEET below for which to pick.
_LAYOUT_MAP = {
    "A8.1": (LAYOUT_CUSTOM, "screenshot"),
    "A8.2": (LAYOUT_CUSTOM, "screenshot"),
    "A8.3": (LAYOUT_CUSTOM, "screenshot_kpi"),
}

# Merges: combine two slides side-by-side into one multi_screenshot slide.
# Format: (left_slide_id, right_slide_id, "Combined Title")
_MERGES = [
    ("A8.5", "A8.6", "Reg E Opportunity: Age Analysis"),
]

# Appendix: these slides are moved out of the main deck into the appendix.
_APPENDIX_IDS = {"A8.7", "A8.12"}

# Skip: these slides are excluded entirely (not even in appendix).
_SKIP_IDS = set()


def register() -> SectionSpec:
    return SectionSpec(
        key="rege",                                        # must be unique
        label="Are Members Opting In to Overdraft Protection?",  # section divider text
        prefixes=_PREFIXES,
        layout_map=_LAYOUT_MAP,
        merges=_MERGES,
        appendix_ids=_APPENDIX_IDS,
        skip_ids=_SKIP_IDS,
    )
```

Then register it in `sections/__init__.py`:

```python
from .my_new_section import register as _my_new_section

SECTION_REGISTRY = [
    ...
    _my_new_section(),   # <-- add here, position = narrative order
    ...
]
```

Done. The assembler picks it up automatically.

## Layout Cheat Sheet

These are the constants you use in `_LAYOUT_MAP`. Import any you need from `._base`.

| Constant | Index | Best For | Notes |
|----------|-------|----------|-------|
| `LAYOUT_CUSTOM` | 8 | **Most data slides** | Title + full-width canvas. Use for any single chart. **This is the default.** |
| `LAYOUT_TWO_CONTENT` | 9 | Side-by-side comparison | Used automatically by merges. Rarely set manually. |
| `LAYOUT_CONTENT` | 2 | Chart with content area | Title + body. Good for charts that need text annotations. |
| `LAYOUT_CONTENT_ALT` | 3 | Same as above, alt colors | Dark background variant. |
| `LAYOUT_PICTURE` | 13 | Chart + commentary | Image left, text right. Good for chart_narrative slides. |
| `LAYOUT_BLANK` | 11 | Full custom | No placeholders. Use when you need total control. |
| `LAYOUT_WIDE_TITLE` | 16 | Wide chart | Similar to CUSTOM but wider title area. |
| `LAYOUT_TITLE` | 1 | Section dividers | Set via `divider_layout` on SectionSpec, not in layout_map. |
| `LAYOUT_SECTION` | 4 | Teal section divider | Alternative divider style. |
| `LAYOUT_TITLE_DARK` | 0 | Dark title slide | For emphasis / intro slides. |
| `LAYOUT_TITLE_RPE` | 17 | RPE branded title | Product-specific. |
| `LAYOUT_TITLE_ARS` | 18 | ARS branded title | Product-specific. |
| `LAYOUT_TITLE_ICS` | 19 | ICS branded title | Product-specific. |

**Rule of thumb:** If your slide is "title + one chart image", use `LAYOUT_CUSTOM`. That covers 90% of cases.

## Slide Type Cheat Sheet

The `slide_type` in the layout map tuple determines how the slide is rendered.

| Slide Type | What It Renders | When to Use |
|------------|----------------|-------------|
| `"screenshot"` | Title + single full-width chart image | **Default for most analysis slides** |
| `"screenshot_kpi"` | Chart on left + KPI callout numbers on right | When you have a chart AND 2-4 key metrics to highlight |
| `"multi_screenshot"` | Two images side by side | Created automatically by merges. Can also set manually. |
| `"mailer_summary"` | 3-column composite (donut, bar, inside numbers) | Mailer-specific. Used for per-month campaign summaries. |
| `"kpi_dashboard"` | Traffic-light colored metric cards (up to 6) | Executive dashboards. Used in preamble. |
| `"kpi_hero"` | 3-4 large KPI numbers + optional chart below | Hero metrics for section openers. |
| `"chart_narrative"` | Chart (60% left) + bullet text (40% right) | When a chart needs explanation text alongside it. |
| `"summary"` | 3x3 grid of bullet points | Section summaries or key takeaways. |
| `"title"` | Title + optional subtitle | Section dividers and title slides. |
| `"section"` | Section divider with large text | Auto-generated by the assembler for each section. |
| `"blank"` | Blank slide with optional title | Placeholders for manual content. |

**Decision tree:**
1. Just a chart? → `"screenshot"` + `LAYOUT_CUSTOM`
2. Chart + 2-4 KPI numbers? → `"screenshot_kpi"` + `LAYOUT_CUSTOM`
3. Chart + text explanation? → `"chart_narrative"` + `LAYOUT_CUSTOM`
4. Just big numbers, no chart? → `"kpi_hero"` or `"kpi_dashboard"` + `LAYOUT_CUSTOM`
5. Two charts side by side? → Use `_MERGES` (auto-creates `"multi_screenshot"`)

## SectionSpec Fields Reference

| Field | Type | Required | Default | Purpose |
|-------|------|----------|---------|---------|
| `key` | str | Yes | — | Unique section identifier. Must match analytics folder name. |
| `label` | str | Yes | — | Question displayed on section divider slide. |
| `prefixes` | list[str] | Yes | [] | Slide ID prefixes that belong to this section. |
| `layout_map` | dict | No | {} | slide_id -> (layout_index, slide_type) |
| `merges` | list | No | [] | Pairs of slides to combine side-by-side. |
| `appendix_ids` | set | No | {} | Slide IDs moved to appendix. |
| `skip_ids` | set | No | {} | Slide IDs excluded entirely. |
| `absorb_ids` | dict | No | {} | Pull slides from another section. Example: `{"A11.1": "value"}` |
| `consolidate` | Callable | No | None | Custom consolidation function. Only needed for complex ordering (mailer). |
| `prefix_fallback` | Callable | No | None | For dynamic slide IDs not in layout_map. |
| `divider_layout` | int | No | LAYOUT_TITLE | Layout for the section divider slide. |

## Examples by Complexity

### Minimal Section (just charts, no special rules)

```python
from ._base import LAYOUT_CUSTOM, SectionSpec

_PREFIXES = ["ics"]

_LAYOUT_MAP = {
    "ICS-1": (LAYOUT_CUSTOM, "screenshot"),
    "ICS-2": (LAYOUT_CUSTOM, "screenshot"),
    "ICS-3": (LAYOUT_CUSTOM, "screenshot_kpi"),
}

def register() -> SectionSpec:
    return SectionSpec(
        key="ics",
        label="Are ICS Accounts Performing?",
        prefixes=_PREFIXES,
        layout_map=_LAYOUT_MAP,
    )
```

### Section with Merges and Appendix

```python
from ._base import LAYOUT_CUSTOM, SectionSpec

_PREFIXES = ["a9", "att"]

_LAYOUT_MAP = {
    "A9.1": (LAYOUT_CUSTOM, "screenshot_kpi"),
    "A9.3": (LAYOUT_CUSTOM, "screenshot"),
    "A9.6": (LAYOUT_CUSTOM, "screenshot"),
    "A9.9": (LAYOUT_CUSTOM, "screenshot_kpi"),
}

_MERGES = [
    ("A9.3", "A9.6", "Attrition: Open vs Closed & Personal vs Business"),
]

_APPENDIX_IDS = {"A9.2", "A9.4", "A9.5"}

def register() -> SectionSpec:
    return SectionSpec(
        key="attrition",
        label="Are We Losing Accounts?",
        prefixes=_PREFIXES,
        layout_map=_LAYOUT_MAP,
        merges=_MERGES,
        appendix_ids=_APPENDIX_IDS,
    )
```

### Section That Absorbs Slides from Another Section

```python
def register() -> SectionSpec:
    return SectionSpec(
        key="dctr",
        label="How Active Are Debit Cards?",
        prefixes=["dctr", "a7"],
        layout_map=_LAYOUT_MAP,
        merges=_MERGES,
        appendix_ids=_APPENDIX_IDS,
        skip_ids={"DCTR-1"},
        absorb_ids={"A11.1": "value"},  # Pull value slide A11.1 into this section
    )
```

## Mapping SLIDE_MAPPING.md to Section Files

The `SLIDE_MAPPING.md` at the repo root documents every module, its chart type, and layout index. When creating a section, use that document as the source of truth:

1. Find your section in SLIDE_MAPPING.md
2. Note the layout index (usually 2 = `LAYOUT_CONTENT`, 8 = `LAYOUT_CUSTOM`, 9 = `LAYOUT_TWO_CONTENT`)
3. Note the chart type to pick the right slide_type
4. Copy the slide IDs into your `_LAYOUT_MAP`

| SLIDE_MAPPING Layout | Section Constant | Typical slide_type |
|---------------------|------------------|-------------------|
| 2 (Title and Content) | `LAYOUT_CONTENT` | `"screenshot"` |
| 8 (Custom Layout) | `LAYOUT_CUSTOM` | `"screenshot"` |
| 9 (Two Content) | `LAYOUT_TWO_CONTENT` | `"multi_screenshot"` |
| 12 (Bullet Points) | `LAYOUT_BULLETS` | `"summary"` |
| 13 (Picture + Content) | `LAYOUT_PICTURE` | `"chart_narrative"` |
